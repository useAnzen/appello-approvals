Interface 1

Banner — Desktop View

A dismissible banner that appears at the top of the dashboard content area. Supports different variants (info, success, welcome) and optional action buttons. Persists dismiss state to localStorage.

Info Banner (Default)

dashboard — info banner
📢
Welcome to Appello v1.33!
We've added Safety & Incident Management, Personnel QR Codes, and improved the Schedule of Values experience. Check out the release notes for full details.
Dev: Component: WelcomeBanner.tsx in apps/frontend/src/components/. Dismiss key stored in localStorage as appello_banner_dismissed_{bannerId}. Banner config fetched from a new SystemBanner model or hardcoded per release.

System

Data Model & Configuration

Banners can be configured per-release or triggered by system events. Admin users can create and manage banners from settings.

Banner Configuration

FieldTypeDescription
idString @idUnique banner ID (nanoid)
titleStringBanner heading text
messageStringBanner body text (supports markdown)
variantEnumINFO | SUCCESS | WARNING | WELCOME
actionLabelString?Optional primary button label
actionUrlString?Optional primary button URL/route
isActiveBooleanWhether the banner is currently displayed
targetAudienceEnumALL | NEW_USERS | ADMINS
dismissibleBooleanWhether users can dismiss the banner
startsAtDateTime?Optional start date for scheduled banners
endsAtDateTime?Optional end date for auto-expiry
releaseVersionString?Linked release version (e.g., "1.33.0")

Dismiss Persistence

// Client-side (localStorage) localStorage.setItem(`appello_banner_${bannerId}`, 'dismissed'); // Server-side (UserPreference model — optional) UserPreference.upsert({ userId: ctx.user.id, key: `banner_dismissed_${bannerId}`, value: 'true' });
Dev: Start with localStorage for MVP. Migrate to server-side UserPreference if cross-device persistence is needed. Check localStorage on component mount to avoid flash-of-banner.
